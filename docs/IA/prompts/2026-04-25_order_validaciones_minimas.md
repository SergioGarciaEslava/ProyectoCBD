# Prompt

El usuario pidio implementar reglas minimas de validacion para `Order` en el proyecto academico RavenShop.

Reglas solicitadas:

- Un pedido no puede crearse vacio.
- Cada `lineItem` debe tener `quantity > 0`.
- El pedido nace en estado `Pending`.
- El total se recalcula siempre en servidor.

Restricciones:

- No crear pantalla de creacion de pedidos.
- No anadir validaciones complejas innecesarias.
- No romper el estilo simple del proyecto.
- No usar autenticacion, JPA ni base relacional.
